## How to use this document

1. **Each module** covers one or more AC content descriptions (codes)
2. **Coverage tracker** shows which codes you have covered so far
3. **Slot modules** into your Schedule tab in Google Sheets
4. **Mix and match** - you don't have to run every module every year
5. **The rule** - by end of Year 8 all K and P codes must be covered across the cohort's two years

## Module Library

These are the teachable units you can slot into any term. Each maps to specific AC codes.

### Year 7/8 Modules

---

#### MODULE: Digital Systems and Hardware
**AC codes covered:** AC9TDI8K01
**Recommended year:** Year 7 (Term 1 or 2) or Year 8 (Term 1)
**Suggested duration:** 3-4 weeks
**What students do:**
- Identify hardware components and explain their role (CPU, RAM, SSD, GPU, I/O devices)
- Compare hardware specifications and make recommendations for use cases
- Explain how operating systems and application software interact with hardware
- Demonstrate how hardware affects performance

**Assessment ideas:**
- System diagram labelling and explanation
- "Build a PC for a purpose" - justify hardware choices for a given task
- Hardware comparison report

---

#### MODULE: Binary Numbers and Data Representation
**AC codes covered:** AC9TDI8K03, AC9TDI8K04
**Recommended year:** Year 7 (Term 2) - you are doing this now
**Suggested duration:** 4-5 weeks
**What students do:**
- Convert between binary and decimal (8-bit)
- Represent text using ASCII
- Represent images using pixels, RGB, colour depth
- Represent audio using sampling, sample rate, bit depth
- Calculate file sizes for images and audio

**Assessment ideas:**
- Binary conversion quiz
- File journey - trace a photo from creation to storage
- "Explain binary to a 5-year-old" video/explainer

---

#### MODULE: File Journeys and Networks
**AC codes covered:** AC9TDI8K01, AC9TDI8K02 (partial)
**Recommended year:** Year 7 (Term 2) - you are doing this now
**Suggested duration:** 2-3 weeks
**What students do:**
- Trace a file from input through processing, storage, transmission and output
- Explain LAN vs WAN, protocols (HTTP/HTTPS), IP addresses, packets
- Describe compression (lossy vs lossless)

---

#### MODULE: Microsoft Office Suite
**AC codes covered:** AC9TDI8P11, AC9TDI8P12 (partial)
**Recommended year:** Year 7 (Term 1) - you did this
**Suggested duration:** 4-5 weeks
**What students do:**
- Use Outlook, OneDrive, Excel, PowerPoint, OneNote effectively
- Apply formulas (SUM, AVERAGE, COUNT), create charts
- Manage files, share documents, understand cloud storage
- Build a digital portfolio

---

#### MODULE: Networks and Cyber Security
**AC codes covered:** AC9TDI8K02, AC9TDI8P13, AC9TDI8P14
**Recommended year:** Year 8 (Term 1)
**Suggested duration:** 4-5 weeks
**What students do:**
- Explain how data travels across networks (TCP/IP, DNS, routers)
- Identify cyber threats: malware, phishing, social engineering, DDoS
- Explain defences: encryption, firewalls, MFA, strong passwords
- Investigate and manage digital footprint
- Assess what data is essential to a system's purpose

**Assessment ideas:**
- Cyber threat analysis report
- "Design a secure school network" task
- Digital footprint audit

---

#### MODULE: Working with Data - Spreadsheets
**AC codes covered:** AC9TDI8P01, AC9TDI8P02, AC9TDI8P03, AC9TDI8P14
**Recommended year:** Year 7 (Term 1 via Excel intro) + Year 8 (Term 2 deeper)
**Suggested duration:** 4-5 weeks
**What students do:**
- Acquire data from surveys, sensors, online repositories
- Clean and validate data in Excel/Sheets
- Apply formulas: VLOOKUP, IF, COUNTIF, pivot tables
- Create charts and visualisations to identify trends
- Build a single-table database and run queries
- Consider data privacy and digital footprint

**Assessment ideas:**
- Data analysis project using real-world dataset (e.g. weather, sports, community data)
- Collaborative data project with a real question to answer

---

#### MODULE: General-purpose Programming
**AC codes covered:** AC9TDI8P05, AC9TDI8P06, AC9TDI8P09
**Recommended year:** Year 8 (Term 2) or Year 7 (Term 2 if students are ready)
**Suggested duration:** 5-6 weeks
**What students do:**
- Design algorithms with nested loops, if/elif/else, functions
- Represent algorithms as flowcharts and pseudocode
- Trace algorithms manually to predict output
- Implement programs in Python (or JavaScript/Scratch as entry point)
- Debug programs systematically

**Assessment ideas:**
- Algorithm design task (flowchart + code)
- Debug challenge - find and fix errors in provided code
- Mini project: build a small working program

---

#### MODULE: Creating a Digital Solution
**AC codes covered:** AC9TDI8P04, AC9TDI8P07, AC9TDI8P08, AC9TDI8P10, AC9TDI8P11, AC9TDI8P12
**Recommended year:** Year 8 (Term 2) - capstone module
**Suggested duration:** 5-6 weeks
**What students do:**
- Interview a "client" or user and create user stories
- Define design criteria
- Generate and evaluate alternative designs
- Build a digital solution (website, program, app, spreadsheet tool, or physical device)
- Test against user stories and evaluate future impact
- Manage the project using agile approaches

**Assessment ideas:**
- Full design portfolio: brief, research, sketches, prototype, evaluation
- Presentation to a real audience ("client")

---

#### MODULE: Digital Footprint and Privacy
**AC codes covered:** AC9TDI8P14
**Can be embedded inside:** Networks and Cyber, Working with Data, Creating a Digital Solution
**Suggested duration:** 1-2 weeks standalone OR woven through other modules
**What students do:**
- Audit their own digital footprint
- Assess which data various systems collect and whether it is essential
- Apply basic privacy principles to their own digital solutions

## The Schedule Tab in your Google Sheet

Once you decide which modules to run, add them to a `Schedule` tab:

| class | term | year | module | start_week | end_week | active | ac_codes |
|-------|------|------|--------|------------|----------|--------|----------|
| Year 7 | 2 | 2026 | Binary Numbers | 4 | 8 | TRUE | K03,K04 |
| Year 8 | 1 | 2026 | Networks and Cyber | 1 | 5 | FALSE | K02,P13,P14 |

The hub reads this tab to know what is current, what is coming, and injects the right content into the AI. Public holiday? Change `start_week` from 4 to 5. Module runs long? Change `end_week`. No rows to hunt down. One cell.
